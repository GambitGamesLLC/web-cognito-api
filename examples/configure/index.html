Cognito API Manager - Configure() Example

This example demonstrates how to use the Configure() method of the CognitoApiManager.

The script for this page fetches the amplifyconfiguration.json file from the parent /examples directory to configure the Amplify library. Check the browser's console for detailed output. The final configuration object loaded into Amplify will be displayed below.

About amplifyconfiguration.json

This JSON file contains the necessary AWS resource details for Amplify to connect to your backend services, like Amazon Cognito.

  • userPoolId: The ID of your Cognito User Pool. You can find this in the AWS Cognito console under "User Pools" > [Your Pool Name] > "Pool details".

  • userPoolClientId: The ID of your App Client. Found in the AWS Cognito console under "User Pools" > [Your Pool Name] > "App integration" > "App clients and analytics".

  • region: The AWS region where your Cognito User Pool is located (e.g., us-east-1). This can also be found in the "Pool details" section of your User Pool in the AWS Cognito console.

  • identityPoolId (Optional): The ID of your Cognito Identity Pool (Federated Identities). This is needed if you plan to use AWS credentials for unauthenticated or authenticated users to access other AWS services. You can find this in the AWS Cognito console under "Identity Pools".

The structure of this file is important. The `web-cognito-api` library expects a format that the AWS Amplify library can understand, typically like this: 

{
  "Auth": {
    "Cognito": {
      "userPoolId": "...",
      "userPoolClientId": "...",
      "region": "..."
    }
  }
}
